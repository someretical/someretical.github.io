This website

That is why this website is so simple. This website consists of a single HTML file, a single CSS file and a few images (no JS at all Woohoo!). I can only hope it loads fine on all the different combinations of user agents (the compatibility tables on MDN only keep growing).

Anyway, I hope you find the simplicity of this website comforting. In my opinion, there are too many over-the-top fancy websites with terrible loading times nowadays.

Gungeon Generator

I've spent a few hundred hours playing Enter the Gungeon and one of the features I love about the game is its intricate dungeon generation system. I found a blog post here that explained how it worked in some detail. While Enter the Gungeon was created with Unity and hence C#, I decided to recreate the dungeon generation system in Python as it was a language I had not seriously used up until that point.

I struggled a lot with implementing the graph algorithms as I had not formally learned graph theory (still haven't) but I got there in the end. Had a wild time Googling and browsing Stack Overflow and I can now declare myself as a somewhat proficient numpy and networkx user :)

Currently, the project is still in the works as I am working on converting the graphs into an ASCII 'bitmap'. This stage is also quite time-consuming but I don't believe it involves any complex maths theory so it's mainly just a LOT of logic that I have to bang my head over.

The repository is here.

Theoretical Diary

This project is my Magnum Opus at the time of writing. I wanted to get some practical experience with C++ after reading Bjarne Stroustrup's official C++ book and decided to make a native desktop app with the Qt framework. The inspiration for this app came from a mobile app called Moodflow that didn't run very smoothly on my phone (probably because my phone wasn't very new).

I spent a bunch of time making the app which turned out pretty nice in the end. The Linux app image is only ~35Mb, while the Windows bundle is a lot larger 85Mb because of the Visual C++ runtime redistributable. Unfortunately, some features don't work on Windows. I had a terrible time unsuccessfully setting up a debug environment on Windows to try and troubleshoot the issue. So if you want all the functionality, you'll have to use the Linux version.

I learnt a lot more than I imagined while building the app. It turned out to be somewhat similar to building a website despite my intentional decision NOT to choose Electron. I can't say if it's harder than building a website since I haven't made a serious website with a database backend, API and all those fancy shenanigans.

Also, learning C++ from a C perspective was quite interesting as it provides so many abstractions but still allows the programmer to directly manipulate memory. It also gave me a deeper insight into how JS interpreters, namely V8 and SpiderMonkey, works and where many JS features originated. Overall, I still feel like I've only touched the surface of C++.

The repository is here.
